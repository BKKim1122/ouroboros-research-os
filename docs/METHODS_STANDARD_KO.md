# 실험 방법론 표준 (모든 세션 공통)

> 목적: 세션(사람이든 다른 Claude든)이 바뀌어도 **같은 방식으로 봉인하고 같은 형식의
> 리포트**가 나오게 한다. V17.5/6/7이 암묵적으로 따른 규약 + 이후 세션 검토에서 확정된
> 개선(null-상대 봉인)을 명문화한 것. 새 실험을 설계·판정하기 전에 이 문서를 따른다.
> 상위 규범은 `INTERP_GUARDRAILS.md`(R1–R5, 자기점검 3문항)이며 이 문서는 그 실무 표준이다.

## 1. 판정 기준(kill criterion) 봉인 방식
1순위 **null-상대 백분위**: 통제 null(라벨 순열 / 공분산 매칭 가우시안 셔플, R≥1000)을
**먼저 구현**하고, 관측치를 null 분포의 백분위로 판정한다. 파일럿 값을 보고 임계를 정하는
것(forking paths)을 원천 차단한다. 예: "지표가 null의 99백분위 초과 AND (대조군 대비
margin ≥ 사전값)".
2순위 **절대 임계**: null-상대가 부적합할 때만. 절대값은 반드시 **선행 확증의 사전등록
값에서 유래**해야 한다(V17.5/6/7이 V17의 0.75/0.65를 재사용한 방식). 즉석에서 지어낸
숫자 금지.
- 어느 방식이든 **결과 관측 전** spec.yaml에 봉인하고, 실행 후엔 **대입만** 한다.
- **양방향 해석**(지지/약화 각각의 뜻)도 결과 전에 spec·코드 출력에 봉인. "약화"와 "반증"을
  구분해 표기(예: 병합=H-A는 약화지 반증 아님).

## 2. 독립 스크립트 표준 (cli/거버너 밖 실험)
`discover.py`·`topk_removal.py`처럼 cli를 안 거치는 스크립트를 **확증**에 쓰려면 반드시:
- **`--mock` 확증 차단**: MockBackend는 truth로 활성치를 직접 만들어 지표가 순환논증으로
  완벽히 나온다(예 ARI≈1.0). mock은 **파이프라인 점검 전용**이며 과학적 의미 0. 확증
  경로에서 mock이면 즉시 중단하도록 가드를 넣는다.
- **null(대조) 구현 필수**: spec의 kill이 참조하는 null·shuffle이 코드에 실제로 있어야
  한다. 없으면 봉인 자체가 불가능.
- **freeze 배선 또는 cli 경유**: 확증 전 코드+프롬프트 해시 동결. 파일럿은 거버너를 안
  거쳐도 되지만, 확증 승격은 freeze + 거버너 판정 경로를 통과해야 한다.
- **seed 분리**(§4).

## 3. 리포트 산출 표준 (형식 고정)
모든 실험은 아래 형식으로 산출한다 — 세션이 바뀌어도 리포트가 동일하게 읽히도록.
- **콘솔**: 지표 표(mean ± sd) → 대조(shuffle/null baseline, 우연선 확인) → `[봉인 판정]`
  한 줄(봉인 라벨 + 근거 수치).
- **JSON**(`results/<name>_report.json`): `{mock, seeds, kill_criterion_sealed(또는
  null 기준), metrics{mean,sd}, verdict_sealed}`.
- **판정 문구**는 코드가 봉인 기준에 대입해 출력하고, 사람이 사후에 고쳐 쓰지 않는다.
- README에 "결과 읽는 법"(어느 지표가 주지표인지, 라벨 의미)을 명시.

## 4. seed 분리
- **파일럿 seed**(파이프라인·null 분포 확정용)와 **확증 seed**(판정용)는 **disjoint**.
- 확증 seed는 선행 파일럿과도 겹치지 않게(예: V17 파일럿 0–2, 확증 3–10, V18 확증은
  20–27 등 신규). 확증 재실행·재선택 금지.

## 5. 거버너 endpoint
- 거버너는 endpoint별 판정 경로를 갖는다(예: `emergence`는 전이/gap 기반 PASS/FAIL).
- **새 유형의 실험**(예: V18 `discovery` = ARI 기반)은 **확증 전에** 해당 endpoint의
  판정 경로를 거버너에 추가한다. 기존 경로에 억지로 매핑하지 않는다.

## 6. 편향 자기점검 (매 판정 필수)
- **"이게 자아·의식 주제가 아니었어도 같은 기준·같은 무게를 적용했을까?"**를 명시적으로
  답해 사용자가 검증하게 한다.
- **자기변호 금지**: "이 좁힘은 결함이 아니라 설계다", "잘 절제돼서 신뢰가 간다" 류의
  중립 기준선 없는 크기딱지(R3 위반)를 결과·논문·리포트에 쓰지 않는다. 배제한 대안을
  **수치로 나열**하는 사실 서술로 대체한다.
- 통제 목록(nuisance·조건)은 실행 전 유한 고정. 실행 후 "하나 더"는 별도 후속 실험이지
  현 판정에 소급 금지(goalpost moving 차단).

## 7. 소스 정합 (세션 인수인계 시)
- 로컬(zip)과 원격(GitHub)이 서로 다른 파일을 앞설 수 있다. 인수인계 전 반드시 대조:
  `git pull --rebase` 전에 로컬 신규 문서가 미커밋인지 확인(유실·충돌 방지).
- 배포 zip은 **필독 문서(`INTERP_GUARDRAILS.md`, `docs/history/`, 이 문서)를 포함한
  완전 스냅샷**이어야 한다. 일부만 담긴 zip은 다음 세션에서 필독 문서 누락을 부른다.

# V16-E.5 Relational Binding Pilot

## 명명 규칙

- 사람용 연구 버전: **V16-E.5**
- 코드/폴더 식별자: `v16e_5`
- 과거 `v0.5` 표기는 사용하지 않는다.

## 목적

V16-E.4의 canonical body embedding 의존성을 제거하고, episode마다 무작위화된 entity symbol과 relation cue의 일치만으로 identity·beneficiary를 binding하도록 한다. factor label용 head, factor-specific slot/gain/route는 사용하지 않는다.

## 핵심 변경

1. entity와 identity/beneficiary relation token에서 canonical body embedding 제거
2. action target을 canonical body가 아니라 현재 제시된 entity slot(0/1)로 정의
3. symbol을 학습된 ID embedding이 아니라 4-bit compositional code로 표현
4. 일부 symbol code를 학습에서 제외하고 OOD symbol로 평가
5. identity×beneficiary×concern 중 `(0,0,0)`, `(1,1,1)`을 학습에서 제외해 조합 OOD 평가
6. allocation/protection rule scalar를 입력하고 held-out rule 조합으로 평가
7. layerwise role/phase patching, cross-factor patch, shuffled donor, random/norm-matched control 수행
8. same-action/different-factor subset에서 factor decodability를 별도로 측정

## 모델

- 하나의 shared Transformer
- 모든 phase에 하나의 shared policy head와 shared outcome head
- factor report head 없음
- factor-specific internal route 없음
- factor-specific loss 없음

## 단계

- continuity: identity-linked memory mismatch 해결
- allocation: beneficiary-linked energy threshold 해결
- protection: concern=1일 때 threshold 이상 위험 보호
- integrated: identity/beneficiary/concern이 결합된 V15형 정책

## Pilot seed

- 16150, 16151, 16152

## 판정 기준

- ID action accuracy: 각 phase 평균 0.95 이상
- symbol OOD, factor-combination OOD: 각 phase 평균 0.85 이상
- rule OOD: allocation/protection 평균 0.80 이상
- designated role+phase patch가 cross-factor, shuffled, random control보다 큼
- world/outcome MAE의 과도한 악화 없음
- same-action/different-factor subset에서 chance 초과 decodability가 여러 seed에서 재현

## 해석 경계

성공하더라도 독립 slot의 자연발생을 의미하지 않는다. 관계 cue가 표면 ID가 아닌 episode-local binding으로 사용되고, 그 정보가 동적 causal routing에 기여함을 지지하는 탐색 결과로 제한한다.

# V16-E.5 진행 보고서

## 1. 버전 명명

기존 파일럿 표기 `v0.5`는 연구계보와 혼동될 수 있어 사람용 버전을 **V16-E.5**로 통일한다. 코드·폴더에서는 `v16e_5`를 사용한다.

## 2. 연구 질문

V16-E.4에서 relation token에 포함됐던 canonical body embedding을 제거한 뒤에도, 하나의 unified behavior-only Transformer가 episode-local symbol binding을 이용해 identity·beneficiary·concern을 행동과 결과예측에 사용할 수 있는가?

내부에는 factor-specific slot, gain, route, report head, factor label loss를 두지 않았다.

## 3. 설계 변경

- entity A/B의 고정 body embedding 제거
- action은 canonical body가 아니라 현재 제시된 entity slot 0/1을 선택
- entity와 relation cue는 공유된 4-bit compositional symbol만 사용
- 일부 symbol code는 학습에서 제외하고 OOD 평가
- `(identity, beneficiary, concern)=(0,0,0),(1,1,1)` 조합은 학습에서 제외
- allocation/protection rule scalar를 입력하고 held-out rule 조합 평가
- seed 16150~16152, 각 650 step

## 4. 기본 성능

3-seed 평균은 다음과 같다.

| 평가 | Continuity | Allocation | Protection | Integrated |
|---|---:|---:|---:|---:|
| ID | 1.000 | 0.995 | 0.999 | 0.997 |
| Unseen symbol | 0.999 | 0.992 | 0.998 | 0.979 |
| Held-out factor combination | 1.000 | 0.994 | 0.999 | 0.994 |
| Held-out rule | 1.000 | 0.879 | 0.885 | 0.997 |

canonical body label 없이도 relation symbol을 entity slot에 binding하여 높은 행동성능을 보였다. unseen symbol 및 미학습 factor 조합에서도 성능이 유지됐다. rule OOD는 통과했지만 ID·symbol OOD보다 낮아 규칙 외삽은 아직 제한적이다.

## 5. Same-action / different-factor 분석

지정 phase의 행동이 factor flip 전후 동일한 사례만 선별했다. 따라서 factor decoding이 단순 action label decoding으로 설명되지 않는 조건이다.

마지막 Transformer block의 designated phase representation에서 3-seed 평균 balanced accuracy:

| Factor | ID | Unseen symbol |
|---|---:|---:|
| Identity | 1.000 | 0.999 |
| Beneficiary | 1.000 | 0.999 |
| Concern | 1.000 | 1.000 |

Integrated phase에서도 identity 0.962/0.947, beneficiary 0.998/0.971, concern 1.000/0.999였다. 따라서 최종 표현의 factor 정보는 현재 행동이 동일한 경우에도 유지되며, unseen symbol로 전이된다.

이는 factor 정보가 단순히 현재 action class와 동일하지 않음을 지지한다. 다만 decodability만으로 인과분화를 의미하지 않는다.

## 6. Relational causal coalition

Layer 1에서 factor counterfactual donor의 다음 위치들을 patch했다.

- relation role token
- 두 entity token
- 해당 phase token
- 이들의 조합

`role + entities + matching phase` 전체 coalition의 counterfactual following은 다음과 같았다.

| Factor | 지정 phase | Integrated phase |
|---|---:|---:|
| Identity | 1.000 | 0.995 |
| Beneficiary | 0.998 | 0.998 |
| Concern | 0.999 | 1.000 |

모든 seed의 최소값도 0.988 이상이었다. 반면 norm-matched random control은 0.027~0.058 수준, shuffled donor는 약 0.279~0.354 수준이었다.

단일 token 효과는 factor와 seed에 따라 달랐다.

- identity는 layer 1에서 role token보다 entity tokens와 phase token에 주로 전달됨
- beneficiary는 role·entity·phase에 분산
- concern은 role+phase coalition이 주 경로

따라서 V16-E.5는 독립 factor slot보다 다음 계산구조를 지지한다.

```text
episode-local relational cue
    -> entity binding
    -> factor별로 상이한 분산 token coalition
    -> phase-specific policy representation
    -> shared action-value geometry
```

## 7. 현재 판정

### 새로 지지되는 내용

1. canonical body embedding 없이도 identity·beneficiary relation을 episode-local symbol equality로 학습할 수 있다.
2. 학습하지 않은 symbol code와 factor 조합에서도 행동과 factor representation이 일반화된다.
3. 행동이 동일한 사례에서도 factor 정보가 phase representation에 남는다.
4. 정합적인 relation/entity/phase coalition patch는 counterfactual 정책을 거의 완전히 전달하고 random·shuffled control과 구별된다.
5. 자연학습되는 인과구조는 단일 factor slot보다 동적·분산 relational routing에 가깝다.

### 아직 지지되지 않는 내용

1. 명시적 과제구조 없이 identity·beneficiary·concern이 자발적으로 생성된다는 주장
2. 모든 seed에서 동일한 단일 factor direction 또는 고정 token 경로가 존재한다는 주장
3. 인간의 자기·알아차림 구조와 직접 동형이라는 주장
4. held-out transition rule에 대한 강한 외삽

## 8. DGX Gate 판정

- symbol/position shortcut 제거: 통과
- held-out factor combination: 통과
- same-action factor representation: 통과
- 정합 causal coalition vs random/shuffled: 통과
- 고정된 단일 factor route: 해당 없음; 분산경로 가설로 수정
- transition rule OOD: 부분 통과

따라서 V16-E.5는 **DGX 확증 설계를 준비할 가치가 있는 강한 파일럿**이지만, 즉시 protocol freeze보다는 마지막 한 단계의 구조적 강건성 검사가 필요하다.

## 9. 다음 단계: V16-E.6

V16-E.6에서는 모델을 더 복잡하게 키우기보다 현재 발견의 대안설명을 제거한다.

1. relation cue의 표면 형태를 완전히 교체하는 cross-encoding transfer
2. symbol bit encoder 변경 또는 무작위 orthogonal code 재학습
3. phase token 명칭·순서 permutation
4. 새로운 state-transition family에서 재학습 및 transfer
5. coalition mediation을 행동효과와 outcome prediction으로 동시 검증
6. 5개 pilot seed에서 최소효과 기준 확인

V16-E.6을 통과하면 DGX용 confirmatory protocol과 exact source snapshot을 동결한다.

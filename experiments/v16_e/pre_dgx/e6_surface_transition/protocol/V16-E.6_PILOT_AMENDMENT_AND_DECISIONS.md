# V16-E.6 Pilot Amendment and Decisions

## 상태

V16-E.6은 확증실험이 아니라 구조 강건성 탐색 파일럿이다. 아래 변경은 모두 결과 확인 전후의 파일럿 조정으로 기록하며, 확증 사전등록으로 취급하지 않는다.

## 단계 A — episode-random normalized code stress test

- episode마다 Rademacher/Gaussian entity code를 새로 생성
- relation/phase token 순서 무작위화
- code norm 약 1
- seed 16160, 800 step

결과: protection은 학습됐지만 identity/beneficiary binding과 held-out factor 조합은 충분히 학습되지 않았다. 이는 자연분화 반증보다 관계 신호의 상대적 크기와 과제 난이도 문제로 판정했다.

감사 한계: 단계 A의 정확한 실행시점 source snapshot은 별도 보존되지 않았다. raw/checkpoint/metadata는 보존한다.

## 단계 B — seed별 독립 orthogonal codebook replication

- 16차원 Hadamard 기반 codebook을 seed마다 column/sign 변환
- 12개 symbol 학습, 4개 symbol 미학습
- relation/phase 순서 무작위화
- seed 16160~16164, 700 step

초기 unit-norm codebook에서는 binding 학습이 약했다. entity/relation code norm을 기존 V16-E.5 bit code와 유사한 크기로 보정한 뒤 재실행했다.

결과: ID 및 held-out factor 조합은 매우 높았으나, 미학습 orthogonal code row로의 전이는 낮았다. 고정 codebook 학습은 relational equality보다 개별 symbol geometry를 부분적으로 암기할 수 있다는 대안설명을 남겼다.

## 단계 C — scale-calibrated episode-random code

- code_dim=16
- 매 episode 새로운 Rademacher/Gaussian code 생성
- code norm을 sqrt(16)=4로 보정
- sparse signed code family를 encoding OOD로 사용
- relation/phase 순서 무작위화
- seed 16165~16169, 800 step

결과: ID, encoding OOD, held-out factor 조합에서 모든 phase가 약 0.99 이상으로 재현됐다. transition-family OOD는 allocation/protection에서 평균 약 0.75~0.77로 부분적이고 seed 변동이 컸다.

## 최종 분석 선택

- 단계 C를 V16-E.6의 주 파일럿 결과로 사용
- 단계 B를 symbol memorization 대안설명 진단으로 보존
- 단계 A를 과도한 low-signal stress condition의 실패 파일럿으로 보존
- transition-family OOD는 통과로 포장하지 않고 다음 단계의 핵심 미해결 문제로 이관
